### 3. Global `⌘K` / `Ctrl+K` keyboard shortcut to focus search

**Problem:** `SearchPage.tsx` line 88 displays `<kbd>⌘K</kbd> jump to search` as a hint in the UI, but no event listener anywhere in the codebase actually binds this shortcut. Pressing `⌘K` does nothing. The shortcut is advertised but non-functional — a broken promise to power users.

**Evidence:** `frontend/src/pages/SearchPage.tsx:88` (hint displayed); `frontend/src/layout/AppNav.tsx` (no `useEffect` / `addEventListener`); `frontend/src/routes/router.tsx` (no global listener); grep of all `.tsx` files finds zero `Ctrl+K` or `Meta+k` handler.

**Proposed solution:** Add a `useEffect` in `AppNav.tsx` that listens for `keydown` and navigates to `/app/search` (which has `autoFocus` on its input at `SearchPage.tsx:81`):

```tsx
// AppNav.tsx — add after imports
import { useNavigate } from 'react-router-dom';

// inside AppNav():
const navigate = useNavigate();
useEffect(() => {
  const handler = (e: KeyboardEvent) => {
    if ((e.metaKey || e.ctrlKey) && e.key === 'k') {
      e.preventDefault();
      navigate('/app/search');
    }
  };
  window.addEventListener('keydown', handler);
  return () => window.removeEventListener('keydown', handler);
}, [navigate]);
```

`AppNav` is mounted on every `/app/*` route, so the shortcut is available everywhere in the app shell. The `autoFocus` attribute on the search input (`SearchPage.tsx:81`) already handles focusing the input on navigation.

**User impact:** Power users (recruiters processing many companies) gain keyboard-first navigation. Reduces friction for repeat users. Converts an advertised but dead feature into a working one.

**Effort:** S
**Confidence:** 10/10

### 5. Recent-searches list in SearchPage (localStorage, last 5)

**Evidence:** `frontend/src/pages/SearchPage.tsx:10–11` (`TRY_CHIPS` constant — static); `frontend/src/pages/SearchPage.tsx:97–109` (idle state renders only `TRY_CHIPS`); `frontend/src/lib/session.ts` (localStorage already used).

**Proposed solution:** Add a `RECENT_KEY = 'cmc.recent'` localStorage slot holding a JSON array of `{ name, number }` (max 5, newest first). Write to it in `SearchPage` whenever a user navigates to a company report (in the `submit` handler and `onOpen` callback in `ResultRow`). Read it on mount and display above the `TRY_CHIPS` when non-empty:

```tsx
// Helpers (can live in format.ts or inline in SearchPage.tsx)
const RECENT_KEY = 'cmc.recent';
const MAX_RECENT = 5;

function getRecentSearches(): { name: string; number: string }[] {
  try { return JSON.parse(localStorage.getItem(RECENT_KEY) ?? '[]'); } catch { return []; }
}
function addRecentSearch(name: string, number: string) {
  const cur = getRecentSearches().filter((r) => r.number !== number);
  localStorage.setItem(RECENT_KEY, JSON.stringify([{ name, number }, ...cur].slice(0, MAX_RECENT)));
}
```

Call `addRecentSearch(hit.companyName, hit.companyNumber)` in `ResultRow`'s `onOpen`. Render in the idle grid before `TRY_CHIPS`:


**User impact:** Returning users (the core repeat-use case for recruiters re-checking shortlist companies) land on a personalised page rather than generic demos. Reduces clicks to re-open frequently checked companies.

### 8. Interactive score calculator on the Methodology page

**Problem:** `MethodologyPage.tsx` (lines 142–158) renders the `RECIPE` array as a static list of score ingredients. The data needed for a live calculator is already present in the `RECIPE` constant (weight strings like `−70`, `+10`, `+5`). Users reading the methodology want to understand how a specific scenario scores — e.g. "what if accounts are overdue AND there's an insolvency case?" — but cannot explore this interactively.

**Evidence:** `frontend/src/pages/MethodologyPage.tsx:25–42` (`RECIPE` constant with weight, heading, paragraph, impact per signal); `MethodologyPage.tsx:142–158` (rendered as a static `.recipe` div; no interactivity).

**Proposed solution:** Add a `ScoreCalculator` component inside `MethodologyPage.tsx` (below the existing `#verdict` section, before the `#confidence` section). The component maintains a `Set<string>` of active signal IDs, parses the weight strings from `RECIPE` (e.g. `'−70'` → `-70`, `'+5 to −15'` → midpoint `-5`), and computes a running total starting from 75:

```tsx
function ScoreCalculator() {
  const [active, setActive] = useState<Set<string>>(new Set());
  const toggle = (h: string) => setActive((prev) => {
    const next = new Set(prev);
    next.has(h) ? next.delete(h) : next.add(h);
    return next;
  });

  const score = Math.max(0, Math.min(100, RECIPE.reduce((acc, r) => {
    if (!active.has(r.h)) return acc;
    // Parse first number from weight string e.g. "−70", "+10", "−5 to −15" → first value
    const n = parseInt(r.w.replace('−', '-').replace('+', ''));
    return acc + (isNaN(n) ? 0 : n);
  }, 75)));

  return (
    <div className="builder-body" style={{ marginTop: 24 }}>
      <h3 style={{ marginBottom: 12 }}>Try it: build a scenario</h3>
      <p style={{ fontSize: 13, color: 'var(--ink-2)', marginBottom: 16 }}>
        Toggle signals on/off to see how the score moves. Starts at 75.
      </p>
      <div style={{ display: 'flex', flexWrap: 'wrap', gap: 8, marginBottom: 20 }}>
        {RECIPE.map((r) => (
          <button
            key={r.h}
            onClick={() => toggle(r.h)}
            className={cn('filter-chip', active.has(r.h) && 'active')}
          >
            {r.w} {r.h}
          </button>
        ))}
      </div>
      <div style={{ marginTop: 12 }}>
        <ScoreGauge score={score} confidence={1} />
      </div>
    </div>
  );
}
```

Import `ScoreGauge` and `cn` (both already used elsewhere). Insert `<ScoreCalculator />` after the static recipe div (line 158). The `RECIPE` constant is file-scoped so no prop passing needed.

**User impact:** Converts a trust-building page into a genuinely useful tool. Users understand the model by doing, not reading. Reduces support friction ("why did my company score X?"). Endole and Creditsafe offer no equivalent transparency.

**Effort:** M (component is small but needs weight parsing logic and style polish)
**Confidence:** 8/10

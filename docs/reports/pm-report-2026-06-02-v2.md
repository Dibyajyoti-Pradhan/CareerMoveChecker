### NEW — Item 4: Pricing page at `/pricing`

**Problem:** The router (`router.tsx`) has no `/pricing` route. If a user types careermove.uk/pricing they get a blank SPA shell. The MarketingNav has no Pricing link. There is no page where Free/Pro/Agency tiers are explained.

**Evidence:**
- `router.tsx` — no `/pricing` route
- `MarketingNav.tsx` — only logo + "Check a company" CTA
- `WaitlistModal.tsx` — Pro/Agency waitlist modal exists and works; it just has no trigger page
- `MarketingFooter.tsx` — no Pricing link

**Proposed solution:**
1. Create `frontend/src/pages/PricingPage.tsx` with three tier cards: Free (current, always free), Pro (waitlist), Agency (waitlist).
2. Wire WaitlistModal for Pro and Agency CTAs (same pattern as existing usage in other pages).
3. Add `/pricing` route to `router.tsx` under `MarketingLayout`.
4. Add "Pricing" link to `MarketingNav.tsx` and `MarketingFooter.tsx`.

**User impact:** Users who want to understand paid features have a destination. Opens the monetisation funnel.
**Effort:** S (new page ~80 lines, 2 routing changes, 2 nav changes)
**Confidence:** 10/10

### NEW — Item 6: Feedback widget on company report page

**Problem:** `api/client.ts:61-67` defines `submitFeedback` with `rating`, `useCase`, and `comment`. The endpoint is wired in the mock API contract. But `CompanyReportPage.tsx` never renders a feedback UI. CareerMove has zero signal on whether users find reports useful, agree with the verdict, or notice errors.

**Evidence:**
- `api/client.ts:61-67` — `submitFeedback` fully defined
- `CompanyReportPage.tsx` — no import or usage of `submitFeedback`
- Competitors: Creditsafe and Endole both collect thumbs-up/down on every result

**Proposed solution:**
1. At the bottom of Zone C (after all tabs), add a compact feedback row: "Was this report useful?" with 👍 / 👎 buttons.
2. On click: call `api.submitFeedback({ companyNumber, rating: 1 or 0, useCase: 'report-page' })`.
3. Show a toast "Thanks for your feedback" on success. Hide the widget after submission (replace with thank-you text).
4. No new API endpoint — uses the existing contract.

**User impact:** First signal from users on report quality. Catches false positives/negatives. Directly informs model improvements.
**Effort:** S (~25 lines in CompanyReportPage.tsx, no new components)
**Confidence:** 9/10

### NEW — Item 7: Fix dead "Export PDF" button in SavedPage → CSV export

**Problem:** `SavedPage.tsx:196` renders `<button className="btn btn-ghost btn-sm"><Icon name="download" /> Export PDF</button>` with **no onClick handler**. Clicking it does nothing. A user who saves 20 companies and wants to export them is left with a dead button.

**Evidence:**
- `SavedPage.tsx:196` — button with `Icon name="download"` and no `onClick`
- `api/client.ts` — no PDF or export endpoint defined
- The mock API contract does not include a PDF endpoint

**Proposed solution:**
1. Replace "Export PDF" with "Export CSV" (more realistic without a backend PDF renderer).
2. Add an `onClick` handler that serialises the saved companies list to CSV (columns: Company Name, Company Number, Note, Saved date) and triggers a download using `URL.createObjectURL(new Blob([csv], { type: 'text/csv' }))`.
3. Show a toast "Exported N companies" on success.
4. No new API endpoint needed — uses the already-loaded `savedList` state.

**User impact:** Makes the export button functional (currently broken UX). Enables users to share/track their saved list in spreadsheets.
**Effort:** S (~20 lines, pure client-side)
**Confidence:** 10/10

### NEW — Item 8: Bulk check CSV export of results

**Problem:** After a bulk check, users get verdicts on-screen but cannot download them. The `BulkCheckPage.tsx` toolbar has filter chips but no export button. Competitors (Creditsafe, Experian) offer batch-result CSV downloads as a core B2B feature.

**Evidence:**
- `BulkCheckPage.tsx:118-123` — toolbar has filter chips only
- `api/client.ts` — no export endpoint
- `BulkResult.rows` type (`types/index.ts`) has all the data: companyName, companyNumber, riskLevel, score, confidence, bucket

**Proposed solution:**
1. Add an "Export CSV" button to the toolbar in `BulkCheckPage.tsx` (visible only when `result` is not null).
2. On click: serialise `result.rows` to CSV (columns: Row, Input, Matched, Company Name, Company Number, Status, Risk Level, Score, Confidence, Bucket) and download.
3. Filename: `careermove-bulk-{uploadId}-{date}.csv`.

**User impact:** Enables the core B2B workflow: upload → check → export for clients. Directly addresses the agency/recruiter persona.
**Effort:** S (~25 lines, pure client-side)
**Confidence:** 10/10

### NEW — Item 10: SearchPage improved no-results empty state

**Problem:** When `SearchPage.tsx` returns 0 results for a query, it renders nothing (the result area is empty). There is no empty-state component, no suggestions, no "try the company number instead", no fallback link to Companies House. Users who get 0 results currently see a blank screen and have no path forward.

**Evidence:**
- `SearchPage.tsx` — `noResults` boolean is computed but only used to render nothing extra
- No empty-state card or suggestions panel
- `TRY_CHIPS` exist in `SearchPage.tsx` for idle state but are hidden on search

**Proposed solution:**
1. When `noResults` is true, render a `state-card` (reusing the existing class from `CompanyReportPage.tsx`) with:
   - "No companies found for '{query}'" heading
   - Tip: "Try searching by 8-digit company number (e.g. 09446231)"
   - Tip: "If the company is very new, it may not appear yet"
   - A "Search on Companies House" fallback link → `https://find-and-update.company-information.service.gov.uk/company-overview?lang=en&q={query}`
2. Keep `TRY_CHIPS` visible in the no-results state so users can try a known-good example.

**User impact:** Reduces dead-ends. Gives users a path forward when their search fails. Improves trust in the product.
**Effort:** S (~20 lines JSX in SearchPage.tsx)
**Confidence:** 10/10
